.Q: Write a function that returns the number of nodes in a binary tree where the nodes valueis greater than or equal some threshold. The binary tree is not organized by value.

int getThresholdCount(binaryTreeNode, int thresholdValue) 

Example binary tree
               20
             /     \
           10       30
          /  \     /  \
         5    35  25   15

Example getThresholdCount(topOfTree, 20) returns 4

A:

typedef struct binaryTreeNode_s {

  unsigned int value;

  struct binaryTreeNode_s* left;

 stuct binaryTreeeNode_s* right;

} binaryTreeNode;

int getThresholdCount(binaryTreeNode *node, int thresholdValue)

{

  int count=0;

  if (NULL==node) {

    return 0;

  }

  if (node->value >= thresholdValue) {

   count++;

  }

 count += getThresholdCount(node->left,thresholdValue) + getThresholdCount(node->right,thresholdValue);

return count;

}

Short problem description: Find and fix mistakes in function that implements a search for value that occurs in an array most often.

Formal definition:

 A[0] = 1
 A[1] = 2
 A[2] = 3
 A[3] = 3
 A[4] = 1
 A[5] = 3
 A[6] = 1

the function may return 1 or 3.

The attached code is still incorrect on some inputs.
Despite the error(s), the code may produce a correct answer for the example test cases.
The goal of the exercise is to find and fix the bug(s) in the implementation.
You can modify at most four lines.

Assume that:
* N is an integer within the range [1..200,000];

* M is an integer within the range [1..10,000];

* each element of array A is an integer within the range [0..M].

Complexity:
* expected worst-case time complexity is O(N+M);

expected worst-case space complexity is O(M), beyond input storage (not counting the storage required for input arguments).

int solution(int M, int A[], int N) {
 int *count = malloc((M + 1) * sizeof(int);
 int i;
 for (i = 0; i <= M; i++)
  count[i] = 0;
 int maxOccurence = 1;
 int index = -1;
 for (i = 0; i < N; i++) {
  if (count[A[i]] > 0) {
   int tmp = count[A[i]];
   if (tmp > maxOccurence) {
    maxOccurence = tmp;
    index = i;
   }
   count[A[i]] = tmp + 1;
  } else {
   count[A[i]] = 1;
  }
 }
 return A[index];
}

Sum to X

Question (Algorithms & Coding): Given an array of integers, and an integer value X, write a function that returns true if 2 elements sum to X.

Solution:

  • Sort the array. Then iterate from both left and right. Since the range is fixed (integer), can use a radix sort (O(n)). Most other sorting algorithms are O(n log n)
  • Iterate over the array, keep a helper set; for each number, check if it is in the set, and if not, add x minus the number to the set. O(n)

Common Mistakes:

  • Get hung up on sorting in O(n log n)

Analysis: Candidate should come up with an O(n) solution with some hinting in the given time. Follow-up question: return true if 4 numbers add up to X. Solution: Create all combinations of 2 numbers (O(n^2)), then fall back to algorithm above. Candidates should at least attempt the follow up, but are not expected to get it correct in time.

Coding: Ask the candidate to code the highest version of the solution that they were able to come up with (2, 3, or 4 elements). The code should be logically correct, with some allowance for syntax errors because of the time limit and coding environment.

Top X

Normal Solution: Go over the data, keep the top x elements in a temp data structure. Use a binary tree or heap for that data structure. O(n*log x)

How to Check if a Binary Tree is Foldable (both sides are mirror images)?

Given an arbitrary binary tree (NOTE not a binary search tree) write a method that takes in the root node of the tree and outputs a boolean indicating whether or not the binary tree is foldable.

Solutions

  • Create a mirror image of the left sub-tree and compare it to the right sub-tree
  • Use a DFS to traverse to traverse both sub-trees, comparing each child branch recursively.
  • Use a BFS to inspect each layer of the binary tree.  Get a list of all nodes (real or null) and inspect for any discrepancies

http://www.geeksforgeeks.org/foldable-binary-trees/

Common Mistakes

  • Not recognizing that a straight in order traversal wont work.  You need to make sure that positions of missing leaves are also mirrored, not just the values.
  • Getting stuck in the weeds of the algorithm after starting to code. Get the candidate to explain their algorithm before they start coding.

Possible Follow-up Questions

  • What are the trade offs between using a DFS and a BFS? (stack frames vs heap memory)

Date Map

How would you design a class that behaves similar to a Java Map <Date,Object> except that get() returns

  • the mapped value if an exact match is present
  • the mapped value of the closest previous Date in the Map if an exact match is not present
  • null if there is no Date in the Map less than or equal to the requested Date

Implement the get() and put() methods of the class. 

Solutions

Show Python BST solution
Show Python dictionary (HashMap) solution

Success Criteria

  • Implements a working solution
  • Avoids O(nlogn) sorting of the backing data structure in order to find nearest entry.
  • Did the candidate get to an efficient solution? (e.g. O(log(n)) insert, O(log(n)) retrieval)

Common Mistakes

  • Progression starts with HashMap, then they realize it isn't sorted. Then List, which is sorted, but has issues with insertion/access time
  • Getting hung up on the O(1) constant time access that a Map usually provides. The key-value put/get semantics are what we care about here, complexity doesn't have to be O(1)
  • Choosing a data store that requires iteration to find the nearest value (e.g. heap)
  • Using a sorted list as the backing store (Lookups will be O(log(n)), but insertions require re-sorting, making insertion O(nlogn)
  • Using a list without random access (C# Lists are LinkedLists)
  • Hung up on date representation (int, long, date object, etc)
  • If doing a tree, assuming that the closest will be 1 level up. It could be further up.

Followup Questions

  • What are good test cases for these two methods?
